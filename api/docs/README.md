## Documentation Standards

When adding new documentation to this directory:

1. **File Naming**: Use UPPERCASE.md for main documentation files
2. **Structure**: Follow the existing format with clear headings and examples
3. **Code Examples**: Include working code examples for all features
4. **Cross-References**: Link to related documentation where appropriate
5. **Keep Updated**: Update documentation when implementing new features

## Contributing to Documentation

- All documentation files should be placed in `/api/docs/`
- Use clear, concise language
- Include practical examples
- Test all code examples before committing
- Update the main README.md when adding new documentation
